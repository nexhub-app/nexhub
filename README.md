# NexHub

> **四合一媒体聚合客户端（动漫 / 漫画 / 小说 / 影视）—— 源即插件 · 共创社区模式的开源阅读 / 观看工具。**

NexHub 是一个基于 [Flutter](https://flutter.dev) 构建的跨平台媒体聚合客户端，把「动漫、漫画、小说、影视」四类内容统一到一个应用里浏览、搜索、阅读与观看。

## 一、项目简介

**NexHub** 的核心设计理念是 **「源即插件 · 共创社区」**：

- **应用只提供「引擎」**，负责界面、阅读器、播放器、网络请求、解析调度等通用能力；
- **「内容从哪来」由「源」决定**。每个源是一份可导入的 JSON 配置文件（可包含声明式选择器，或内嵌 JavaScript 脚本），描述如何从一个具体站点抓取列表、详情、章节与内容；
- **应用不内置、不打包任何预备源**。所有源由用户自行导入，或在社区中共享。任何人都可以自己写一份源 JSON，导入后即可解析该站点的内容，**无需改动任何应用代码**。

这种设计的收益：

1. **合法与安全**：公开仓库不包含任何指向具体站点的「内置源」，降低版权与合规风险；
2. **可持续与开放**：解析能力由社区共同维护，站点改版只需更新对应源文件，应用本体保持稳定。

> ⚠️ 正因为「不内置源」，首次安装后的 NexHub 是「空的」——你需要导入至少一个源，才能看到内容。这就像买了一个书架，但书要自己放上去。

## 二、核心特色

### 1. 四合一内容聚合
在一个应用内统一管理「动漫、漫画、小说、影视」四类内容，提供统一的浏览、搜索、收藏与历史体验。

### 2. 源即插件 · 共创模式（核心）
- **解析逻辑与站点解耦**：每个源是一份 JSON，描述如何抓取某个站点；
- **声明式 + 脚本式两种写法**：
  - 声明式：用 XPath / JSONPath / CSS 选择器描述抽取规则；
  - 脚本式：在源内嵌 JavaScript（由 `flutter_js` 沙箱执行），处理动态渲染、登录态、加密参数等复杂场景；
- **多引擎调度 `ResolverRegistry`**：根据源的 `parser.type` 自动分发到 `builtin / xpath / jsonpath / css / script / webview / hybrid` 等解析器；
- **不写死任何站点**：应用代码中没有任何硬编码的源地址，全部通过可导入的源文件驱动。

### 3. 首页多板块与动态筛选
- 源可声明「首页板块」（`homeSections`），否则自动根据分类生成多板块；
- 支持「标签筛选」等动态筛选，筛选选项直接取自源定义的真实站点分类，做到「所见即所得」。

### 4. 漫画阅读器
支持双页 / 单页、翻页手势、选话、阅读进度、左右滑动切换等，已针对「区间超屏、图片缺失、进度回跳」等常见问题做过专项修复。

### 5. 小说阅读器
支持字体、字号、行距、边距、主题、亮度等样式自定义，设置即时生效并持久化；控件带有动画反馈，二级设置以内联面板形式呈现，不打断阅读。

### 6. 影视播放器
基于 `media_kit` + `fvp` 的视频播放能力，配合 `canvas_danmaku` 弹幕渲染；针对播放线路选择、集数去重、播放界面等做过优化。

### 7. 下载管理
支持内容离线下载、下载设置与任务管理，便于在无网络环境下阅读 / 观看。

### 8. 云同步（占位）
设置中已预留「云同步」入口，当前为占位实现，后续可由社区或官方接入具体同步方案。

### 9. RSS 订阅
内置可配置的 RSS 路由，可订阅如动漫资讯等公开 RSS 源。

### 10. 历史与更新追踪
记录浏览历史，并追踪章节更新时间（针对部分不返回更新时间的源做了兜底）。

### 11. Material 3 设计 + 动态配色
基于 Material 3 设计系统，支持亮 / 暗主题、自定义主色，并在支持的系统中启用 Material You 动态取色。

### 12. 多语言
内置中文 / 英文语言包（基于 Flutter 官方 `gen-l10n`），用户可见文案走国际化，便于全球社区协作。

### 13. 跨平台
基于 Flutter，具备编译到 **Android / iOS / Windows / macOS / Linux / Web** 的跨平台能力。

## 三、用户使用指南

### 3.1 获取与安装
- **从源码构建**：参见下方[开发者指南](#四开发者指南)（推荐开发者）。
- **使用预编译包**：从项目的 Releases 页面下载对应平台安装包（如 Android APK、Windows 安装程序）。具体发布渠道以仓库 Releases 为准。

### 3.2 首次启动与导入源（关键）
由于本应用**不内置任何源**，首次启动后是空的，需要自行导入：

1. 在应用内找到「源管理 / 导入源」入口；
2. 粘贴一份源 JSON，或选择本地的源 JSON 文件；
3. 导入成功后，该源对应的站点内容即可在首页 / 分类中浏览，**无需改动应用代码**。

> 源 JSON 的字段规范与架构基线，参见 [`docs/ARCHITECTURE_BASELINE.md`](docs/ARCHITECTURE_BASELINE.md)。

### 3.3 日常使用
- **浏览 / 搜索**：在首页或分类中浏览，使用搜索按关键词检索；
- **阅读 / 观看**：点击进入详情，选择章节开始阅读漫画 / 小说，或播放影视；
- **下载**：对感兴趣的内容使用离线下载，便于无网络时查看；
- **筛选**：部分源提供标签 / 分类筛选，可快速定位内容。

### 3.4 常见问题
- **为什么看不到任何内容？** 多半是还没导入源。请先按 3.2 导入至少一个源。
- **导入源后仍是空白？** 检查源是否适用于当前网络环境，或该站点是否需要登录态 / 反爬处理（可尝试由源作者更新源脚本）。
- **仓库里没有源，去哪找？** 源由社区共享维护；请通过社区渠道获取并自行评估其合规性与内容合法性。

## 四、开发者指南

### 4.1 环境准备
- 安装 [Flutter SDK](https://flutter.dev)（要求 `>=3.4.0`，`<4.0.0`）；
- 配置好对应平台的编译环境（如 Android Studio / Xcode / Visual Studio 等）；
- 确认 `flutter doctor` 无明显报错。

### 4.2 获取与运行代码
```bash
git clone https://github.com/nexhub-app/nexhub.git
cd nexhub
flutter pub get
flutter run
```
选择目标设备（模拟器 / 真机 / 桌面端）即可启动。

### 4.3 构建发布包
```bash
# Android
flutter build apk
# Windows
flutter build windows
# 其它平台以此类推
```

### 4.4 项目结构
```
nexhub/
├── lib/                  # 应用主体代码（Dart）
│   ├── core/             # 核心能力：网络请求、解析调度、源仓库、阅读器/播放器基础设施
│   │   ├── models/       # 数据模型（如 PluginConfig 源配置）
│   │   ├── resolver/     # 解析引擎调度（builtin / script / webview / hybrid）
│   │   ├── scraper/      # 列表/详情/章节抓取与抽取
│   │   └── services/     # 源仓库、历史、下载等服务
│   ├── features/         # 各业务模块：首页、源管理、漫画、小说、影视、设置、下载……
│   └── l10n/             # 多语言资源（中文 / 英文）
├── assets/               # 资源（图标、通用热搜词等；不含任何站点源）
├── plugins/              # 源目录（见下方说明）
│   └── builtin/          # ⚠️ 此目录在仓库中仅保留占位，不提交任何源 JSON
├── docs/                 # 架构基线说明（仅 ARCHITECTURE_BASELINE.md）
├── test/                 # 单元测试（含源 JSON 校验，无内置源时自动跳过）
├── android/ ios/ linux/ macos/ windows/ web/   # 各平台工程
└── pubspec.yaml          # 依赖与资源声明
```

**关于 `plugins/builtin/`**：本仓库**不会**把任何 `.json` 源文件提交进去（已在 `.gitignore` 中排除）。该目录仅以 `.gitkeep` 占位，保证工程结构完整、可正常构建。真正的源由你保存在本地、或来自社区共享，**不进入公开仓库**。

**已被排除、绝不入库的内容**：

- `plugins/builtin/*.json`（任何内置 / 预备源）
- `reference/`（源的本地备份与参考样例）
- 内部开发文档、重开发提示词与计划、测试数据扫描脚本等（仅保留架构基线说明 `docs/ARCHITECTURE_BASELINE.md`）
- `build/`、`sdk/`（构建产物与 Flutter SDK，体积巨大）
- `.workbuddy/`、`.trae/`、`.qoder/`、`.probe_tmp/`（本地工具与探针临时产物）
- 根目录的临时脚本（如 `tmp_*.mjs`、`run_probe.*`、`net_check.*`）

### 4.5 源即插件：如何编写与贡献一个源
NexHub 的解析能力完全由源 JSON 驱动。一个源是一个 JSON 文件，描述如何从一个站点抓取内容。其结构（权威规范见 [`docs/ARCHITECTURE_BASELINE.md`](docs/ARCHITECTURE_BASELINE.md)）：

- **标识与站点**：包含 `id`、`name`、站点 `baseUrl`、`version` 等元信息；
- **解析规则**：
  - 声明式：通过 `selectors`（CSS / JSONPath / XPath）描述列表、详情、章节、内容等的抽取位置；
  - 脚本式：通过 `parser.overrides` 内嵌 JavaScript（在 `flutter_js` 沙箱中执行），处理动态渲染、登录态、加密参数等复杂场景；
- **解析器类型**：`parser.type` 决定用哪种引擎——`builtin / xpath / jsonpath / css`（声明式）、`script`（内嵌 JS）或 `hybrid`（按路由混合）；
- **首页与筛选**：可声明 `homeSections`（首页板块）与 `filters`（筛选组），否则由应用按分类自动生成。

**最小示例（示意，字段以架构基线文档为准）**：
```json
{
  "id": "demo_manga",
  "name": "示例漫画源",
  "version": 1,
  "baseUrl": "https://example.com",
  "parser": { "type": "xpath" },
  "selectors": {
    "category": {
      "categories": [{ "id": "hot", "name": "热门" }]
    },
    "list": { "items": "//div[@class='item']/a", "title": "./span" },
    "detail": { "title": "//h1", "chapters": "//div[@class='chapter']/a" }
  }
}
```

**贡献方式**：
1. 在自己的分支编写源 JSON，导入应用验证抓取是否正确；
2. 如需分享，通过社区渠道发布你的源 JSON（**请勿将源提交到本应用仓库**——仓库只托管引擎）；
3. 想改进引擎 / 阅读器 / 播放器 / 文档，欢迎提交 Issue 与 Pull Request。

> 共创的前提是**合规与尊重版权**：只解析你有权访问、且允许抓取的公开内容；不要将源用于侵犯他人合法权益的场景。

### 4.6 测试
```bash
flutter test
```
测试套件包含源 JSON 校验等；当本地没有内置源时，相关用例会自动跳过（契合开源无源模式）。
