## 一、项目简介

**NexHub** 的核心设计理念是 **「源即插件 · 共创社区」**：

- **应用只提供「引擎」**，负责界面、阅读器、播放器、网络请求、解析调度等通用能力；
- **「内容从哪来」由「源」决定**。每个源是一份可导入的 JSON 配置文件（可包含声明式选择器，或内嵌 JavaScript 脚本），描述如何从一个具体站点抓取列表、详情、章节与内容；
- **应用不内置、不打包任何预备源**。所有源由用户自行导入，或在社区中共享。任何人都可以自己写一份源 JSON，导入后即可解析该站点的内容，**无需改动任何应用代码**。

这种设计的收益：

1. **合法与安全**：公开仓库不包含任何指向具体站点的「内置源」，降低版权与合规风险；
2. **可持续与开放**：解析能力由社区共同维护，站点改版只需更新对应源文件，应用本体保持稳定。

> ⚠️ 正因为「不内置源」，首次安装后的 NexHub 是「空的」——你需要导入至少一个源，才能看到内容。这就像买了一个书架，但书要自己放上去。

## 二、核心特色（重点详解）

NexHub 与同类工具最大的不同，在于它把「解析能力」彻底从应用里剥离出来，交给社区用源文件维护。下面把几项**最具特色、也最容易让人误解**的能力展开说明。

### 2.1 源即插件 · 共创模式（核心中的核心）

这是 NexHub 的灵魂，请优先理解：

- **解析逻辑与站点完全解耦**：应用代码里没有任何「为某个站点写死的逻辑」。一个站点能不能被解析，只取决于你有没有一份对应的源 JSON；
- **声明式 + 脚本式两种写法，自由组合**：
  - **声明式**：用 XPath / JSONPath / CSS 选择器描述「列表项在哪、标题在哪、章节链接在哪」。适合结构稳定的服务端渲染页面，零代码、改起来直观；
  - **脚本式**：在源里内嵌一段 JavaScript（在 `flutter_js` 安全沙箱中执行），用来处理动态渲染、登录态注入、加密参数拼接、接口签名等复杂场景；
- **多引擎调度 `ResolverRegistry`**：应用读到一个源后，会按它的 `parser.type` 自动选择解析器——
  - `builtin` / `xpath` / `jsonpath` / `css` → 声明式解析；
  - `script` → 交给 JS 沙箱执行源内脚本；
  - `hybrid` → 按路由在「声明式 / 脚本式」之间智能切换；
  - `webview` → 借助内嵌浏览器渲染后再抽取；
- **同一个引擎，无限个源**：社区每天都能新增、修正源，而应用本身几个月不更新都照样能用。

> 一句话：**应用是书架，源是书。书架不挑书，书也不挑书架。**

### 2.2 源 JS 沙箱的运行铁律（写源的人必读）

源内脚本运行在 `flutter_js`（QuickJS 内核）沙箱中，与 Dart 主线程隔离。这里有两条**经过真机验证**的重要约束，理解它们能少走很多弯路：

1. **异步通道是受限的**：沙箱对 `Promise` / `async` 的支持并不完全。需要发起 HTTP 请求拿数据时，源不应自己 `await fetch`，而是返回一个「协议对象」告知引擎「去哪个 URL 取数据、取回后用哪个函数处理」，由 Dart 侧统一抓取后再把数据回灌给同步处理函数；
2. **同步处理、桥接取数**：把「计算」和「取数」分开——取数交给引擎的桥接通道，源脚本只做纯同步的解析与转换。这套「通用协议」对纯同步脚本零影响，对需要联网的脚本则提供唯一可靠通道。

掌握这两点，你就能在源里实现「登录态抓取、接口签名、动态分页」等高级效果，而不会卡在「为什么 async 不生效」。

### 2.3 首页多板块与动态筛选（所见即所得）

为了让不同源的首页体验一致又不雷同：

- **首页多板块**：源可以声明 `homeSections`（自己定义首页要展示哪些板块、各板块叫什么）；如果源没声明，应用会**自动**根据源的分类生成「最新 + 各分类」多个板块（自动跳过「全部 / 汇总」这类聚合项，最多 8 块），让旧源无需重新导出就能立刻拥有多板块首页；
- **动态筛选**：源可以声明 `filters.groups` 来提供筛选维度。其中**分类本身已经是顶部 Tab，不会重复变成筛选组**，避免「筛选栏和分类栏长得一样」的尴尬；当源提供标签路由 + 标签清单时，应用会自动生成「标签筛选」，且选项的 `value` 用的是站点真实的标签 slug（例如古风站的 `gufeng` 而非带连字符的 `gu-feng`），`label` 用中文显示名——保证筛出来的内容**准确对得上站点**。

> 这套机制的关键教训是：**先确认 URL / slug 拼写，再判断页面是不是 JS 渲染**。曾经有源因为标签 slug 拼错（多了连字符）导致返回空壳页，被误判为「SPA 客户端渲染」而走了搜索兜底接口，结果搜索是全文标题匹配、内容不准。拼写正确后，服务端渲染页直接就能抓，内容才准确。

### 2.4 漫画阅读器（专项打磨）

不是「能看」就行，而是针对真实痛点做过专项修复：

- 支持**双页 / 单页**两种版式，翻页手势、选话（选章节）、左右滑动切换；
- 专门处理过「**区间超屏**」（超长条漫跨屏）、「**图片缺失**自动跳过占位」、「**进度回跳**」（阅读进度被错误重置）等常见问题；
- 阅读进度与书签持久化，关掉再打开能回到原处。

### 2.5 小说阅读器（六条体验原则）

小说阅读器遵循一组明确的体验原则，确保「设置真的生效、不互相打架」：

1. **样式可共存**：字体、字号、行距、边距、主题、亮度等样式以「组合 / 位标志」共存，而不是单选覆盖——你开了护眼又开了夜间，二者同时生效；
2. **即时生效 + 持久化**：改任何设置立刻看到效果，并写入本地，下次打开还是你调的样子；
3. **每一项都真生效**：不摆样子，开关之间不冲突；
4. **控件有反馈**：调节时有动画 / 视觉反馈，知道「动了」；
5. **二级设置内联**：进阶设置用「内联 / 底部可预览面板」呈现，**不单独开一页**，不打断阅读流；
6. **改动先问**：涉及 UI / 交互的改动，先给方案确认再做。

> 另外：阅读器中间偏左的亮度调节，对应「左侧竖向滑动手势」，与视频播放器的交互习惯保持一致。

### 2.6 影视播放器（含弹幕）

- 基于 `media_kit` + `fvp` 提供跨平台硬解播放能力；
- 配合 `canvas_danmaku` 做**弹幕渲染**，营造「一起看」的氛围；
- 针对「播放线路选择、集数去重、播放界面交互」做过专项优化，减少「选了播不了 / 集数重复 / 切集卡顿」等糟心问题。

### 2.7 下载管理

支持内容离线下载、下载设置与任务管理，方便在无网络（地铁、飞机、流量贵）环境下继续阅读 / 观看。

### 2.8 云同步（占位）

设置中已预留「云同步」入口，当前为占位实现，后续可由社区或官方接入具体同步方案（如 WebDAV、自建同步服务等）。

### 2.9 RSS 订阅

内置可配置的 RSS 路由，可订阅如动漫资讯等公开 RSS 源，在应用内直接阅读更新。

### 2.10 历史与更新追踪

记录浏览历史，并追踪章节更新时间（针对部分不返回更新时间的源做了兜底，避免「永远显示未更新」）。

### 2.11 Material 3 设计 + 动态配色

基于 Material 3 设计系统，支持亮 / 暗主题、自定义主色，并在支持的系统中启用 Material You 动态取色（自动跟随壁纸色调）。

### 2.12 多语言

内置中文 / 英文语言包（基于 Flutter 官方 `gen-l10n`），**所有用户可见文案都走国际化**，方便全球社区协作与本地化。

### 2.13 跨平台

基于 Flutter，具备编译到 **Android / iOS / Windows / macOS / Linux / Web** 的跨平台能力（具体平台支持以各版本构建为准）。

## 三、用户使用指南

### 3.1 获取与安装
- **从源码构建**：参见下方[开发者指南](#四开发者指南)（推荐开发者）。
- **使用预编译包**：从项目的 Releases 页面下载对应平台安装包（如 Android APK、Windows 安装程序）。具体发布渠道以仓库 Releases 为准。

### 3.2 首次启动与导入源（关键）
由于本应用**不内置任何源**，首次启动后是空的，需要自行导入：

1. 在应用内找到「源管理 / 导入源」入口；
2. 粘贴一份源 JSON，或选择本地的源 JSON 文件；
3. 导入成功后，该源对应的站点内容即可在首页 / 分类中浏览，**无需改动应用代码**。

> 源 JSON 应该长什么样、有哪些字段，见下方[开发者指南 · 4.5 源 JSON 字段规范](#45-源-json-字段规范)。

### 3.3 日常使用
- **浏览 / 搜索**：在首页或分类中浏览，使用搜索按关键词检索；
- **阅读 / 观看**：点击进入详情，选择章节开始阅读漫画 / 小说，或播放影视；
- **下载**：对感兴趣的内容使用离线下载，便于无网络时查看；
- **筛选**：部分源提供标签 / 分类筛选，可快速定位内容。

### 3.4 常见问题
- **为什么看不到任何内容？** 多半是还没导入源。请先按 3.2 导入至少一个源。
- **导入源后仍是空白？** 检查源是否适用于当前网络环境，或该站点是否需要登录态 / 反爬处理（可尝试由源作者更新源脚本）。
- **仓库里没有源，去哪找？** 源由社区共享维护；请通过社区渠道获取并自行评估其合规性与内容合法性。

## 四、开发者指南

### 4.1 环境准备
- 安装 [Flutter SDK](https://flutter.dev)（要求 `>=3.4.0`，`<4.0.0`）；
- 配置好对应平台的编译环境（如 Android Studio / Xcode / Visual Studio 等）；
- 确认 `flutter doctor` 无明显报错。

### 4.2 获取与运行代码
```bash
git clone https://github.com/nexhub-app/nexhub.git
cd nexhub
flutter pub get
flutter run
```
选择目标设备（模拟器 / 真机 / 桌面端）即可启动。

### 4.3 构建发布包
```bash
# Android
flutter build apk
# Windows
flutter build windows
# 其它平台以此类推
```

### 4.4 项目结构
```
nexhub/
├── lib/                  # 应用主体代码（Dart）
│   ├── core/             # 核心能力：网络请求、解析调度、源仓库、阅读器/播放器基础设施
│   │   ├── models/       # 数据模型（如 PluginConfig 源配置）
│   │   ├── resolver/     # 解析引擎调度（builtin / script / webview / hybrid）
│   │   ├── scraper/      # 列表/详情/章节抓取与抽取
│   │   └── services/     # 源仓库、历史、下载等服务
│   ├── features/         # 各业务模块：首页、源管理、漫画、小说、影视、设置、下载……
│   └── l10n/             # 多语言资源（中文 / 英文）
├── assets/               # 资源（图标、通用热搜词等；不含任何站点源）
├── plugins/              # 源目录（见下方说明）
│   └── builtin/          # ⚠️ 此目录在仓库中仅保留占位，不提交任何源 JSON
├── test/                 # 单元测试（含源 JSON 校验，无内置源时自动跳过）
├── android/ ios/ linux/ macos/ windows/ web/   # 各平台工程
└── pubspec.yaml          # 依赖与资源声明
```

**关于 `plugins/builtin/`**：本仓库**不会**把任何 `.json` 源文件提交进去（已在 `.gitignore` 中排除）。该目录仅以 `.gitkeep` 占位，保证工程结构完整、可正常构建。真正的源由你保存在本地、或来自社区共享，**不进入公开仓库**。

**已被排除、绝不入库的内容**：

- `plugins/builtin/*.json`（任何内置 / 预备源）
- `reference/`（源的本地备份与参考样例）
- 内部开发文档、重开发提示词与计划、测试数据扫描脚本等（整个 `docs/` 目录均不入库）
- `build/`、`sdk/`（构建产物与 Flutter SDK，体积巨大）
- `.workbuddy/`、`.trae/`、`.qoder/`、`.probe_tmp/`（本地工具与探针临时产物）
- 根目录的临时脚本（如 `tmp_*.mjs`、`run_probe.*`、`net_check.*`）

### 4.5 源即插件：如何编写与贡献一个源

NexHub 的解析能力完全由源 JSON 驱动。一个源是一个 JSON 文件，描述如何从一个站点抓取内容。其结构与字段见下方 **[4.5 源 JSON 字段规范](#45-源-json-字段规范)**。

- **标识与站点**：包含 `id`、`name`、站点 `baseUrl`、`version` 等元信息；
- **解析规则**：
  - 声明式：通过 `selectors`（CSS / JSONPath / XPath）描述列表、详情、章节、内容等的抽取位置；
  - 脚本式：通过 `parser.overrides` 内嵌 JavaScript（在 `flutter_js` 沙箱中执行），处理动态渲染、登录态、加密参数等复杂场景；
- **解析器类型**：`parser.type` 决定用哪种引擎——`builtin / xpath / jsonpath / css`（声明式）、`script`（内嵌 JS）或 `hybrid`（按路由混合）；
- **首页与筛选**：可声明 `homeSections`（首页板块）与 `filters`（筛选组），否则由应用按分类自动生成（详见 2.3）。

**最小示例（声明式漫画源）**：
```json
{
  "id": "demo_manga",
  "name": "示例漫画源",
  "version": 1,
  "baseUrl": "https://example.com",
  "parser": { "type": "xpath" },
  "selectors": {
    "category": {
      "categories": [{ "id": "hot", "name": "热门" }]
    },
    "list": { "items": "//div[@class='item']/a", "title": "./span" },
    "detail": { "title": "//h1", "chapters": "//div[@class='chapter']/a" }
  }
}
```

**贡献方式**：
1. 在自己的分支编写源 JSON，导入应用验证抓取是否正确；
2. 如需分享，通过社区渠道发布你的源 JSON（**请勿将源提交到本应用仓库**——仓库只托管引擎）；
3. 想改进引擎 / 阅读器 / 播放器 / 文档，欢迎提交 Issue 与 Pull Request。

> 共创的前提是**合规与尊重版权**：只解析你有权访问、且允许抓取的公开内容；不要将源用于侵犯他人合法权益的场景。

### 4.5 源 JSON 字段规范

> 这是源文件的权威速查表（原先在仓库 `docs/` 中的架构基线说明，现已内联于此，方便贡献者无需额外文档即可编写源）。字段以应用实际解析为准；下表覆盖最常用的核心字段。

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | string | 源的唯一标识（同名导入时按 `version` 决策覆盖 / 跳过） |
| `name` | string | 源的展示名称 |
| `version` | int | 版本号；导入同名源时：`>=` 已安装版本则替换（含同版本重导入以应用编辑），`<` 已安装版本则跳过 |
| `baseUrl` | string | 站点根地址 |
| `parser.type` | string | 解析引擎：`builtin` / `xpath` / `jsonpath` / `css` / `script` / `webview` / `hybrid` |
| `selectors` | object | 声明式抽取规则（按 `parser.type` 选用 CSS / JSONPath / XPath 语法） |
| `selectors.category.categories` | array | 分类列表，每项 `{ "id": "...", "name": "..." }`；用于自动生成首页板块 |
| `selectors.list` | object | 列表页规则：`items` / `title` / `cover` / `link` 等子选择器 |
| `selectors.detail` | object | 详情页规则：`title` / `chapters` / `cover` 等 |
| `parser.overrides` | string(JS) | 脚本式解析逻辑（JS 源码字符串），在 `flutter_js` 沙箱执行 |
| `homeSections` | array | 可选；自定义首页板块。未声明则按分类自动生成 |
| `filters.groups` | array | 可选；自定义筛选组。`category` 维度的筛选会被自动剔除（分类已是 Tab） |
| `tagSearch` | object | 可选；标签检索路由，配合 `selectors.category.tags` 生成标签筛选；`value` 用站点真实 slug |

**关于 `version` 的导入规则（务必注意）**：
- 源作者发新版，只需把 JSON 里的 `version` 调大；
- 导入同名（`id`）源时：版本 **≥ 已安装版本** → 替换（高版本升级 / 同版本重新导入以应用你的编辑）；版本 **< 已安装版本** → 跳过，防止误装旧版冲掉新源；
- 内置源不可被覆盖；导入时会保留原有的「启用 / 隐藏」状态。

### 4.6 测试
```bash
flutter test
```
测试套件包含源 JSON 校验等；当本地没有内置源时，相关用例会自动跳过（契合开源无源模式）。
